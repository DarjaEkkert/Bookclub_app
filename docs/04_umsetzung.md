## Ergebnis
- Der initiale komplexe Prompt führte zu keinem erfolgreichen Build.
- Daher wurde die Entwicklung schrittweise in kleinere Teilprompts aufgeteilt.
- Jeder Prompt wurde dokumentiert (siehe docs/02_prompts.md).
- Auf diese Weise konnte der Entwicklungsprozess strukturiert nachvollzogen werden.

## Version 3 – Bearbeiten und Löschen

Die Anwendung wurde um folgende Funktionen erweitert:
- Bücher können bearbeitet werden (Edit-Funktion)
- Bücher können gelöscht werden (Delete-Funktion)

Damit wurde ein vollständiges CRUD-System umgesetzt.
